## Overview

End-to-end OSINT investigation of an active Instagram-based food delivery scam targeting Brazilian consumers. The operation used inflated follower counts, a well-crafted order landing page, and PIX-only payment to execute financial fraud at scale — with evidence of **at least 11 parallel profiles** using the same template and infrastructure.

The investigation was triggered by an X post with 50+ comments from users reporting the same scam. From there, the attacker's Instagram profile, web infrastructure, and related campaigns were fully mapped using open-source tools.

## How the Scam Works

```
Instagram Sponsored Story
         │
         ▼
  Victim clicks link
         │
         ▼
 Order website (brutusburguerapp[.]com)
  ┌──────────────────────────────────┐
  │  • "Someone is buying right now" │
  │  • Countdown timer (urgency)     │
  │  • Location: any city in Brazil  │
  │  • Payment: PIX only             │
  └──────────────────────────────────┘
         │
         ▼
  Victim pays via PIX → order never arrives
```

**Psychological tactics used:**
- **Authority** — franchise branding, large follower count
- **Urgency** — countdown timers, limited-time promotions
- **Social proof** — fake real-time purchase notifications
- **Trust signals** — "Secure Environment" banners

## Web Page Analysis

**Domain:** `brutusburguerapp[.]com`  
**IP:** `34.174.128.95` (Google Cloud Platform, US)

The order site was well-crafted and convincing:

- Location selector accepts **any city in Brazil** — in every case, the nearest franchise is listed as exactly 4.5km away
- Displays fake real-time purchase notifications
- Promotional countdowns to create urgency
- Collects full name and delivery address
- **Only accepts PIX** — untraceable by ordinary users

**WAF Evasion:**  
The application's WAF blocked:
- Burp Suite proxy connections
- VPN connections
- Session cookie access

> This may be a default hosting configuration, but may also have been intentionally configured to prevent forensic analysis of the PIX QR code generated by the app.

## Recommendations

- **Block and report** the identified IPs and domains to hosting providers
- **Contact domain registrars** to initiate takedown of active scam domains
- **Deploy social media monitoring** tools to automatically flag template-based scam profiles
- **Educate users** to verify franchise legitimacy before making PIX payments to unfamiliar sites
- **Further investigation:** WAF bypass via Burp Suite or isolated VM to capture and analyze the PIX QR code payload
